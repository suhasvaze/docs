# 1. COA Values, Combinations etc

# 1. Values

  1. COA Values have additional attributes like Summary, Allow Posting, Allow Budgeting, Account Type, Third Party Control A/c etc. These attributes appear only when the COA is deployed. Hence enter/import values only after Deployment.

  2. Summary = "Yes" Indicates a Parent Value and Summary = "No" indicates a Child Value

  3. Only Child Values can be used while entering Account Combinations (Not the Parent Values)

  4. For Child Values, set Allow Posting = "Yes". This will allow Journal Posting. Also set Allow Budgeting="Yes" so that Budgeting can be done on these

  5. For Parent Values, set Allow Posting = "No" and Allow Budgeting="No"

  6. Some attributes are available ONLY for Account Segment. E.g. Account Type (Owners Equity, Asset, Liability, Expense, Revenue, Budgetary Debit and Budgetary Credit) and Third Party Control A/c (No, Customer, Supplier, TPCA).




# 2. Segment Labels/Qualifiers

  1. Segment Labels in Fusion are attached to Segments while creating COA Structure (Primary Balancing Segment, Natural Account, Cost Center etc) -- These were referred as Flexfield Qualifiers in EBS (These should not be confused with Tree Labels of Account Hierarchy)

  2. COA Value Attributes E.g. Allow Posting, Allow Budgeting, Third Party Control A/c etc. -- these do not have any specific terminology in Fusion. They can be referred just as Segment Value Attributes. These were referred as Segment Qualifiers in EBS.




# 3. Account Combinations

Task: Manage Account Combinations

When Segment Values are joined together, it forms a Account Combination (also called as GL Code Combinations)

E.g.

  1. 110-4501-92250 or 110.4501.92250 are examples of GL Code Combinations (also called as Account Combinations)
  2. A full Account Combination must be specified while entering a Journal Line
  3. All previously used Account Combinations are stored in gl_code_combinations table. A Code Combination ID (called CCID or Account ID) is generated/assigned for each combination stored in this table.

# 4. Dynamic Combinations

Task: Manage Chart of Accounts Structure Instance

While creating Chart of Account Structure Instance it allows to define a check box

[X] Dynamic Combination Creation Allowed

When checked,

  1. it allows to use (e.g. while Entering Journals) New Valid A/c Combination which is not previously used and not previously stored in gl_code_combinations table

  2. The New Valid A/c Combination is automatically inserted in gl_code_combinations table and a CCID is generated and stored in it. This is also referred as Dynamic Insertion




# 5. Segment Values Inheritance

Process: Inherit Segment Value Attributes

  1. When the characteristics/attributes of values in the value sets are updated, such as changes to enabled status, effective date, posting allowed status, or natural account type, all previously created account combinations that referenced such values aren't automatically updated by these changes.

  2. The Inherit Segment Value Attributes process lets you run a controlled process to update such existing account combinations.

Example

A/c Value 12190 has Account Type as "Asset". Some A/c Combinations are already created as follows

  • 110.10.12190.310.000.000 - Account Type = "Asset"
  • 230.20.12190.115.000.000 - Account Type = "Asset"
  • 410.10.12190.210.000.000 - Account Type = "Asset"

Now Account Type for 12190 is changed to "Liability", then such change needs to be reflected in above A/c Combinations also. The Inherit process updates Account Combinations with such changes.




# 6. Custom Segment Labels

Task: Manage Chart of Accounts Structure
         (Search for General Ledger and Go to Actions→Manage Segment Labels)

  1. Seeded Segment Labels are Primary Balancing Segment, Natural Account, Cost Center etc
  2. Custom Segment Labels can have custom names and they can be used to during SLA Rules Customisation

Most of the implementations will NOT need Custom Segment Labels. These may be needed for Advanced Usage and Customization of SLA Accounting Rules.




# 7. Account Aliases

Task: Manage Shorthand Aliases

  1. [X] Shorthand alias enabled should be checked for the relevant COA Structure Instance in task Manage Chart of Account Structure Instances
  2. Shorthand Aliases are names given for Full or Partial A/c Combinations that are frequently used
  3. When Shorthand Alias is selected in Journal Line, it populates the segment values as per the Alias Setting
  4. Job of entering full A/c Code Combinations can be simplified by using Shorthand Aliases
  5. Deploy COA if you change this setting.

Example

Shorthand Alias A/c Combination
Travel Expenses 101.10.60501.121.000.000
Freight 101.10.62510.000.000.000

While entering A/c for Journal Line, if we select Travel Expenses alias, that will populate all COA Segments as above.
